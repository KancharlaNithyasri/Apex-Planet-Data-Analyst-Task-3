# Task 3: Deep-Dive Analytics & Interactive Dashboarding

## Project Overview
This repository contains the complete documentation, technical data pipeline, and interactive deliverables for **Task 3** of the Data Science Internship at **Apex Planet Software Solutions Pvt Limited**. 

Using the `Sample - Superstore.csv` dataset, this project establishes a formal KPI framework, executes an advanced customer behavioral segmentation using Python, and deploys an interactive executive dashboard in Power BI to optimize regional retail profitability.

---

## 🛠️ Project Pipeline & Phases

### 🔹 Step 1: Metric Framework Definition
Before data processing, 4 core Key Performance Indicators (KPIs) were formally defined to track the financial health of the Superstore. This step involved drafting the business rationale along with the explicit mathematical formulas and technical DAX logic:
*   **Total Sales:** Gross revenue generated by customer orders.
*   **Overall Profit Margin (%):** Efficiency metric tracking net profitability.
*   **Average Order Value (AOV):** Measuring the average monetary value per unique transaction.
*   **Product Return Rate:** Operational leakage metric tracking order rejection frequency.

### 🔹Step 2: Deep-Dive Customer Segmentation (RFM Analysis)
To move past basic data summaries, an advanced **RFM (Recency, Frequency, Monetary)** behavioral framework was developed in Python using `pandas`:
*   **Recency (R):** Days elapsed since a customer's last transaction relative to the dataset snapshot baseline.
*   **Frequency (F):** Total number of unique orders placed by a single customer.
*   **Monetary (M):** Total lifetime value (gross spend) generated per customer.

Using custom mathematical quartile boundaries, customers were grouped into actionable business segments including **Champions (VIPs)**, **At Risk Loyalists**, **New Buyers**, and **Lost Customers** to drive targeted marketing and discount-minimization strategies.

### 🔹 Step 3: Interactive Power BI Dashboard Build
The final phase translated backend data modeling into frontend business intelligence. A 1-to-many relational model was established between the transactional data and the RFM segment tables inside **Power BI Desktop** to build a dynamic executive interface.
*   **Core Scorecards:** High-level KPI tiles for immediate corporate tracking.
*   **Segment Revenue Treemap:** Instant visual breakdown of revenue shares per behavioral tier.
*   **Profitability Matrix:** Column charts tracking profit margins across segments to find margin drains.
*   **Interactive Slicers:** Dynamic regional and categorical filtering.
